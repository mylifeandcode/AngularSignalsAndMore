Signal Inputs

We can now create inputs that are signals. This is done with the input function. These inputs are read-only. 

value = input(0);

You bind to them the same way as the original style @Input()s. 

<custom-slider [value]="50" />

The type is inferred from the default value, or you cans specify it with a generic (more readable). 

value = input<number>(); //Type is actually number|undefined since we aren't providing a default value

You can make an input required. 

value = input.required<number>();

You can also provide configuration for an input to do things like transformation and aliasing (should be avoided). 

label = input('', {{'{'}}transform: trimString{{'}'}});

You can find the official docs, which go into more detail, here.
